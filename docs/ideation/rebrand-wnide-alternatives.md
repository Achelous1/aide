# wnide 계열 대체 후보 탐색

> `rebrand-wnide.md` 에서 도출된 "카테고리 자체를 질문/부정하는" 브랜드 톤을 유지하면서
> wnide 의 구조적 리스크(whine 동음, wn- 자음 클러스터, 백크로님)를 회피하는 대안들을 탐색합니다.

## 1. 요약

- wnide 의 **톤(도발/질문/반카테고리)** 은 유지, **매개체(이름 자체)** 는 교체.
- 가장 유력한 3후보: **noide**, **yide**, **voide**.
- 최종 권고: **noide** — 메시지 직관성·발음·네임스페이스 3축 모두 양호. wnide 의 whine 리스크 없음.

## 2. "비슷한 느낌" 의 정의

wnide 에서 계승할 브랜드 속성:

| 속성 | 설명 |
|---|---|
| 도발성 | IDE 카테고리 자체에 대한 질문 혹은 부정 |
| 짧음 | 5자 이하, 한 음절~두 음절 |
| 소문자 네이티브 | `ripgrep`, `nvim`, `curl` 톤 |
| `ide` 접미 보존 | 카테고리 내에서 자기 정체성 선언 |
| 언더그라운드 | 기업형 블렌드가 아닌 해커 톤 |

wnide 에서 **버릴** 속성:

| 속성 | 이유 |
|---|---|
| `wn-` 자음 클러스터 | 영어에 존재하지 않는 onset → 발음 불가 |
| "whine" 동음 | 제품 포지셔닝 정반대 |
| 백크로님 구조 | "Why Need IDE?" 의 영문법 비문성 |

## 3. 브레인스토밍 풀

카테고리별 후보. ⭐ = 쇼트리스트.

### A. 의문형 (Interrogative)

| 후보 | 해석 | 1차 평가 |
|---|---|---|
| ⭐ **yide** | "Y IDE?" (Y=Why, 코더 네이티브 약어) | 짧고 의미 직관, `yide.dev` 네임스페이스 유력 |
| **ynide** | "Y Need IDE?" wnide 의 Y 버전 | wnide 의 `wn-` 문제 해결, 다만 여전히 백크로님 |
| **whide** | "Why IDE?" 압축 | **"wide"(넓은)** 와 혼동 — 탈락 |
| **qide** | "Q IDE?" (Q=query) | 해석 불명확 |

### B. 부정·반대 (Negational)

| 후보 | 해석 | 1차 평가 |
|---|---|---|
| ⭐ **noide** | "no IDE" — 안티-IDE 선언 | 가장 직관적, 발음 명료 /ˈnoʊ.aɪd/ |
| **unide** | un- + IDE | 발음 /juːˈnaɪd/ "united" 유사 → 혼동 |
| **anide** | anti- + IDE | 발음 /ˈæn.aɪd/, 철자 전달 약함 |
| **xide** | X out IDE | X- 접두사 과용 피로 (X.com, Xcode, X…) |
| **antide** | antidote + IDE | **기존 약물명 충돌** (GnRH 길항제) |
| **endide** | end + IDE | 네거티브 과잉, 제품 수명 암시 |

### C. 공백·여백 (Void / Bare)

| 후보 | 해석 | 1차 평가 |
|---|---|---|
| ⭐ **voide** | void + IDE — 빈 캔버스 은유 | 시적, "void"(빈) 의미가 조립 철학과 일치 |
| **rawide** | raw + IDE — 날것 | 발음 /ˈrɔːaɪd/ 어색, "rawhide"(생가죽) 연상 |
| **baride** | bare + IDE | "baride" 미확정 발음, 스페인어 "varide" 와 혼동 |
| **zide** | zero + IDE | 의미 연결 약함 |

### D. 소유권 (Ownership)

| 후보 | 해석 | 1차 평가 |
|---|---|---|
| **ownide** | own + IDE — "네 IDE 는 네가 가져" | 메시지 강하나 /oʊnˈaɪd/ 발음 복잡 |
| **myide** | my + IDE | 너무 유치·범용 |
| **uride** | ur-/your + IDE | "uride" 어원 불명 |

### E. 반어·선언 (Declarative)

| 후보 | 해석 | 1차 평가 |
|---|---|---|
| **byeide** | bye + IDE — "IDE 안녕" | 유머러스, 제품이 IDE 를 대체한다는 선언 위배 |
| **exide** | ex- + IDE | **Exide Technologies 상표 충돌**(배터리) — 탈락 |
| **notide** | not + IDE | "notide" 가 "tide(조류)" 와 시각적 충돌 |

## 4. 쇼트리스트 3 후보 상세

### 4.1 noide — "no IDE"

- **의미**: 기성 IDE 를 거부한다는 직접 선언. 조립형 환경이 곧 "no IDE" 상태로 시작함을 드러냄.
- **발음**: /ˈnoʊ.aɪd/ "NO-eyed" — 영어·한국어·일본어 모두 명료.
- **네임스페이스 예측**: `noide.dev`, `noide.sh`, npm `noide`, `github.com/noide` — 유력 미선점 (정식 조회 필요).
- **상표 리스크**:
  - "No IDE" 는 일반명사 조합 → 상표성 약할 수 있음 (진입 장벽 ↓, 방어력 ↓).
  - "Noide" 조어로 출원 시 상표성 확보 가능.
- **약점**:
  - **포지셔닝 함정**: "IDE 가 아니다" 를 과하게 강조하면 실제 IDE 기능(터미널·에디터·에이전트)을 제공함에도 부정해야 함.
  - 서사 자원이 단순 — "no" 이상의 확장 스토리 부족.
- **태그라인**: "noide — the IDE you didn't install."

### 4.2 yide — "Y IDE?"

- **의미**: 가장 짧은 질문형. Y 는 코더 문화에서 "Why" 의 자연스러운 약어 (`git log -y`, `y/n` 프롬프트).
- **발음**: /jaɪd/ "yide", 한 음절. 4글자 최단.
- **네임스페이스 예측**: `yide.dev` — 4자 도메인은 희소 → 선점 경쟁 가능성. npm `yide` 는 미선점 가능성 높음.
- **상표 리스크**:
  - 4자 조어는 출원 난이도 ↓.
  - "yield"(수확·양보) 와 발음 유사 — 금융·농업 SEO 노이즈 가능.
- **약점**:
  - 짧음이 **과도한 축약** 으로 느껴질 수 있음.
  - 의미 전달이 "Y=Why" 전제에 의존 — 비개발자에게 불투명.
- **태그라인**: "yide — because the question was never rhetorical."

### 4.3 voide — "void + IDE"

- **의미**: 빈 공간(void)이 곧 출발점. 플러그인으로 채워 완성되는 캔버스 은유.
- **발음**: /vɔɪd/ 혹은 /ˈvɔɪd.aɪd/ — **"void" 와 정확히 동음** 이 될지, `-ide` 까지 발음할지 모호.
- **네임스페이스 예측**: `voide.dev` — 미선점 가능성 중간 (void 파생어는 보안·게임 업계에 일부 존재).
- **상표 리스크**:
  - **"Void Linux"** 존재 — 개발자 툴 카테고리 인접, 주의 필요.
  - **"Voidtools"** (Everything 검색기) — Class 9 활동.
  - 음성 유사성 소송 리스크 Partide §6 수준으로 평가 필요.
- **약점**:
  - Partide 의 "particle" 같은 화학 서사 대신 "void" 의 실존적 서사 — 호불호 분명.
  - 한국어권에서 "보이드" 는 게임/수학 용어로 먼저 인지 → 제품 카테고리 혼동.
- **태그라인**: "voide — start from nothing. assemble anything."

## 6. 권고

1. **1순위: noide** — wnide 의 도발 톤을 계승하면서 발음·의미 직관성을 모두 확보. 서사 확장을 위해 "no + IDE" 를 "naked + IDE" / "node IDE" 등으로 재해석 가능한 여지까지 덤으로 제공.
2. **2순위: yide** — 극한 미니멀리즘이 브랜드 가치라면 선택. 4자 도메인 선점이 가능할 때에 한정.
3. **3순위: voide** — 서사·시적 깊이를 우선할 경우. 단 Void Linux 등 인접 상표 clearance 필수.
4. **Partide 와의 관계**: 이 세 후보 모두 Partide 를 대체하기보다, **철학 서브브랜드 / CLI 별칭 / 캠페인 슬로건** 으로 공존시키는 안이 유력:
   - Product: **Partide**
   - CLI/hacker alias: `noide` 혹은 `yide`
   - Manifesto title: "voide" 혹은 "Why need an IDE?"
